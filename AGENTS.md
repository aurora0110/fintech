# Collaboration Principles

- 请使用第一性原理思考。
- 不能总是假设用户非常清楚自己想要什么和该怎么得到。
- 请保持审慎，从原始需求和问题出发；如果动机和目标不清晰，先停下来讨论。
- 如果目标清晰但是当前路径不是最短，明确指出，并建议更好的办法。
- 当用户提出多个任务并且确认执行后，先主动分析任务之间的依赖关系、先后顺序和潜在冲突，再给出建议执行顺序；在没有新约束阻塞的情况下，按确认后的顺序主动推进。
- 请保持与用户的沟通活跃，及时反馈进度和结果，避免用户因为等待而感到焦虑。
- 如果有不知道、不确定以及不理解的内容，请明确告知用户，不要乱编或者胡乱回答。
- 在写代码的时候，要注意代码的质量和可维护性，同时减少重复计算、冗余代码，使用高效的计算方式。
- 在回答问题的时候，可以发散思维，考虑不同的角度、方向和维度，提供更全面、更有深度的回答。
- 不要轻易停止任何一个正在运行中的程序，除非得到用户的明确确认。
- 在需要创建新代码的时候，除非用户明确指定在某个文件夹下创建新的代码文件，否则都在/Users/lidongyang/Desktop/Qstrategy/utils/tmp下创建新的代码文件。

## Backtest And Research Guardrails

- 在启动任何大规模回测前，先检查参数是否完整；只要存在关键交易定义缺失，例如持有天数、买入价、卖出价、止盈止损触发方式，就先停下来确认，不能靠默认值偷偷补齐。
- 在声称“结果已跑完”之前，必须同时确认两件事：进程已经结束，且结果文件时间戳已经更新到本轮运行时间。不能只根据进度条、数据加载完成或代码实现完成来判断。
- 对于长时间运行的实验，必须优先保存中间结果，至少包括每组参数的日收益序列、净值序列或其他可复算核心指标的中间产物，避免汇总指标出错后被迫整轮重跑。
- 任何账户层指标，尤其是年化收益率、最终净值、最大回撤，必须基于真实净值序列计算，不能把单日收益率序列误当作净值序列。
- 输出回测结果前，必须做基本的结果合理性校验；如果出现如下情况，视为结果无效，先排查再汇报：最大回撤小于 `-100%`、最终净值小于等于 `0`、年化收益率与净值方向明显矛盾。
- 如果结果文件里保存的只是汇总指标而不是时间序列，必须明确告知用户这些结果是否可复算；不能在无法复算的情况下承诺“稍后修一下指标就行”。
- 当同一脚本可能被重复启动时，必须先检查是否已有旧进程在运行；如果用户要求只保留最新任务，必须显式识别并清理旧进程，避免多个同名实验同时写同一结果目录。
- 在需要停止进程时，优先按命令行精确匹配具体脚本，而不是模糊杀掉所有 Python 进程。
- 当实验时间明显超出预期时，要尽早给出全量穷举耗时估算，并主动提出分阶段搜索、缓存复用或缩小参数空间的替代方案。
- 对于组合回测，必须明确说明账户层口径：初始资金、最大持仓数、选股排序规则、是否允许同日重叠、收益是按单笔还是按账户净值统计，避免用户把不同口径混为一谈。
- 任何“最佳参数”结论都必须区分单笔层和账户层；如果账户层指标有 bug 或未验证，只能报告“单笔领先候选”，不能报告“最终最优策略”。
- 修改回测脚本后，先做最小可验证检查，例如语法检查、组合数校验、少量样本 sanity check，再启动全量实验，避免数小时后才发现低级错误。
- 每轮实验默认写入独立结果目录，目录名应带时间戳或参数签名，避免新旧结果混淆。
- 监控长任务时，不能只看终端输出；要同时检查进程存活、结果文件时间戳和中间产物是否更新。
- 必须明确区分单笔统计、信号日篮子近似和真实账户回测，不能混用口径下结论。
- 任何买卖、止盈、止损规则都必须先落成清晰时间轴：`signal_date`、`entry_date`、`first_eligible_exit_date`。
- 排序因子必须使用个股当天实际值，不能误用参数阈值常数参与排序。
- 同一交易日如果止盈和止损同时触发，必须预先固定优先级，不能临时解释。
- 触发价格和实际成交价格必须分开处理，尤其是次日开盘执行时要考虑跳空。
- 评估最佳参数时，除了样本数，还要关注交易日覆盖范围和时间分布，避免被少数年份主导。
- 参数搜索后的结论默认视为可能过拟合；重要结论应尽量补样本内外或滚动验证。
- 最大连续失败次数必须注明统计口径；逐笔连续失败不等于账户净值连续回撤期。
- 中断实验后重新运行时，要么切换结果目录，要么先清理或归档旧结果，避免状态污染。
- 全量实验前，优先先跑小样本冒烟回测，验证交易数、净值、回撤和执行时点是否合理。
- 本项目后续所有回测默认剔除 `2015-06-01` 到 `2024-09-30` 之间的数据，除非用户明确要求保留该区间。

## Strategy Semantics

- 趋势线和多空线的交易语义默认按以下理解：
  - 强票通常不会有效跌破趋势线；即使盘中回踩趋势线，也应能较快收回。
  - 收盘价在多空线之上，通常意味着仍有资金控盘、结构未坏。
  - 若连续三天收盘位于多空线下方，且后续再有一天价格跌破这三天中的最低价，可视为控盘减弱、出货概率升高、结构走坏。
  - 研究和回测中，趋势线更偏短期强弱分界，多空线更偏中期控盘/结构分界；两者的破坏语义不能混用。
