# Qstrategy 回测公共约定

## 适用场景

- 用户要比较两个以上策略。
- 用户要确认正式口径、研究口径、观察口径。
- 用户要定义统一买卖时间轴、统计口径、优先级。

## 口径分层

### 正式口径

- 交易定义稳定
- 用户认可可作为 baseline
- 买点、卖点、执行方式都固定

### 研究口径

- 用来验证假设
- 可临时扩参数、扩模型、扩特征
- 结果不能直接当正式策略

### 观察口径

- smoke
- 极小样本
- 单一阶段统计
- 只用于发现方向

## 默认优先级

除非用户另行指定，Qstrategy 当前默认目标优先级是：

1. 买点成功率
2. 上涨概率
3. 平均单笔收益
4. 账户层净值

## 时间轴

每次回测必须明确：

- `signal_date`
- `entry_date`
- `first_eligible_exit_date`

默认规则：
- 买入当日不能卖
- 触发价格和成交价格必须分开
- 同日止盈止损同时触发时，必须预先固定优先级

## 输出要求

至少说明：

- 口径类型：正式 / 研究 / 观察
- 单笔 / 信号日篮子近似 / 真实账户层
- 数据区间
- TP/SL 触发和成交方式
- 是否有样本不平衡或时间分布问题

## 使用方法

当用户提到：

- “这个实验算正式吗”
- “口径统一一下”
- “这两个结果能不能直接比较”

就先使用本 skill。
